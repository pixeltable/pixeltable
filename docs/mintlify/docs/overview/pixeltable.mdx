# Pixeltable -- Open Source AI Data Infrastructure

Tired of choosing between months of infrastructure work or limited frameworks? Pixeltable gives you both speed and control. Pixeltable provide a **declarative**, **incremental** approach for **multimodal** AI workloads.

<CodeGroup>
```python Declarative
# Persistent and versioned tables holding data
t = pxt.create_table('image_table', {'image': pxt.Image})

# Express data transformations, model inference, and custom logic
t.add_computed_column(detections=yolox(images.image,model_id='yolox_s'))

# Insert data from any source to run inference (URL, S3, Files, Pandas, XML...)
t.insert([
    {'image': 's3://bucket/image.jpg'},
    {'image': 'https://example.com/image.png'},
    {'image': '/local/path/image.jpg'}
])

```

```python Multimodal
# Efficiently manage text, images, video, and audio in a unified interface
t = pxt.create_table(
    "multimdodal_table",
    {
      'document': pxt.Document,   # For HTML, MD, .txt, PDF, XML documents
      'video': pxt.Video,         # For video files
      'audio': pxt.Audio,         # For audio files
      'question': pxt.String,     # User queries
      'timestamp': pxt.Timestamp, # For DateTime values
      'metadata': pxt.Json,       # For lists or dictionaries
      'embedding': pxt.Array[(512,), pxt.Float]
      ... and more
    }
)
```

```python Incremental
# Pixeltable handles data orchestration from data to model outputs
chat_table.insert([{
                    'user_id': user_id,
                    'question': message.content,
                    'timestamp': datetime.now()
                }])
result = chat_table.select(
           chat_table.response).order_by(chat_table.timestamp,            
           asc=False).limit(1).collect()

response = result['response'][0]
```
</CodeGroup>
