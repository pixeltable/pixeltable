## Installation

```bash
pip install 'pixeltable[cli]'
```

The `[cli]` extra pulls in `fastapi[standard]` (which provides `uvicorn`). The pcli client itself is stdlib-only; the extra is required only to run the daemon.

Verify the installation:

```bash
pcli --help
pcli health
```

On the first command, pcli auto-spawns a daemon bound to `127.0.0.1:22090`. The daemon survives across shells and stays warm for subsequent commands. Override with `PCLI_PORT`.

## Command structure

```text
pcli <command> [args...]
```

Use `pcli <command> --help` for per-subcommand flags and examples.

| Category | Commands |
|---|---|
| **Inspection** | `ls`, `describe`, `columns`, `computed`, `idxs`, `history`, `status`, `config` |
| **Query** | `rows`, `get`, `count`, `errors` |
| **Mutation** | `drop`, `rm`, `rename`, `mv`, `revert` |
| **Interactive** | `shell` |

### Universal flags

These flags work the same way across the commands that support them and are not repeated in the per-command tables below.

| Flag | Available on | Description |
|---|---|---|
| `-h`, `--help` | Every command | Print the command's usage and examples |
| `--json` | Every command with a human-formatted default output | Emit machine-readable JSON instead. Not accepted by `shell` (interactive REPL) or `health` (always JSON). |
| `-n`, `--dry-run` | Every mutation (`drop`, `rm`, `rename`, `mv`, `revert`) | Print the intended action; don't execute |
| `-f`, `--force` | Mutations that need confirmation (`drop`, `rm`, `revert`) | Skip the `[y/N]` prompt. Required in non-interactive contexts |

## Quick reference

```bash
# inspect
pcli ls -l                          # everything under root, with metadata
pcli ls some_dir --counts           # row counts (parallelized)
pcli describe my_dir/my_table       # schema
pcli rows my_dir/my_table -n 5      # first 5 stored cells

# query
pcli get my_dir/my_table 42         # PK lookup
pcli count my_dir/my_table          # row count
pcli errors my_dir/my_table         # rows where a computed column failed

# ops
pcli drop my_dir/my_table -f
pcli mv my_dir/my_table other_dir
pcli revert my_dir/my_table --steps 3 -f

# interactive
pcli shell
```

## Inspection commands

### `pcli ls`

List entries under a directory.

| Flag | Description |
|---|---|
| `-l`, `--long` | Include column count, last version, and flags |
| `--counts` | Include row counts (runs `count()` per table, parallelized) |
| `--tree` | Tree view |

```bash
pcli ls                            # root
pcli ls some_dir                   # contents of some_dir
pcli ls -l some_dir                # with metadata
pcli ls --counts                   # with row counts
```

Output of `pcli ls -l`:

```text
path                kind   cols  version  flags
agent_demo          dir                       -
audio_chunks        view      9        3  ci
chess_vids          table     3        3  ci
chk                 table     2        1  i
```

Flag letters: `c` = has at least one computed column, `i` = has at least one index.

### `pcli describe`

Show a table's schema and metadata. The plain form is human-readable; `--json` returns the full `get_metadata()` dict.

```bash
pcli describe my_dir/my_table
pcli describe my_dir/my_table --json
```

### `pcli columns` / `pcli computed`

List columns across one or all tables. `pcli computed` is shorthand for `pcli columns --computed`.

| Flag | Description |
|---|---|
| `--computed` | Restrict to computed columns (no effect for `pcli computed`, which always sets it) |

```bash
pcli columns                       # every column in the catalog
pcli columns my_dir/my_table       # one table
pcli columns --computed            # computed columns across every table
pcli computed                      # same as above
```

### `pcli idxs`

List indexes. Shows both B-tree and embedding indexes by default; the `--embedding` flag restricts to embedding indexes.

| Flag | Description |
|---|---|
| `--embedding` | Restrict to embedding indexes |

```bash
pcli idxs                          # every index
pcli idxs my_dir/my_table          # indexes on one table
pcli idxs --embedding              # only embedding indexes
```

### `pcli history`

Show a table's version timeline.

| Flag | Description |
|---|---|
| `-n N` | Show at most N most recent versions |

```bash
pcli history my_dir/my_table
pcli history my_dir/my_table -n 5  # last 5 versions
```

### `pcli status`

Daemon and runtime state: pxt version, daemon PID, configured paths, total tables, total errors.

| Flag | Description |
|---|---|
| `--sizes` | Also report media and file-cache disk usage (slower; scans the directories) |

```bash
pcli status
pcli status --sizes --json
```

### `pcli config`

Every documented configuration setting with its current value and source (`env`, `file`, or `unset`). Credentials show `<redacted>` when set; the `source` column reveals presence even when the value is masked.

```bash
pcli config
pcli config --section openai
pcli config --source env
```

## Query commands

### `pcli rows`

Show the first N rows of a table. Unstored computed columns are skipped by default (selecting one forces evaluation, which can invoke LLMs or expensive compute); pass them explicitly via `--cols` to include them.

| Flag | Description |
|---|---|
| `-n N` | Number of rows (default 10) |
| `--cols a,b,c` | Comma-separated column subset. Default: all stored columns |

```bash
pcli rows my_dir/my_table -n 3
pcli rows my_dir/my_table --cols id,text,score
```

### `pcli get`

Look up a single row by primary key. A numeric-looking PK token is coerced to int or float; everything else stays a string. There is no quoting escape for a string-typed PK whose value looks numeric — if your PK column is a string but the value is `42`, the server will reject the type mismatch. The table must declare a primary key. Unstored computed columns are skipped unless requested explicitly via `--cols` (consistent with `rows`).

| Flag | Description |
|---|---|
| `--cols a,b,c` | Comma-separated column subset. Default: all stored columns |

```bash
pcli get my_dir/my_table 42                    # single-column PK, int
pcli get my_dir/my_table some_string_id        # single-column PK, string
pcli get my_dir/my_table 42 abc                # composite PK, in declared order
pcli get my_dir/my_table 42 --cols id,text     # restrict to listed columns
```

### `pcli count`

```bash
pcli count my_dir/my_table         # prints the integer
pcli count my_dir/my_table --json
```

### `pcli errors`

List rows where a stored computed column failed. The table must have a primary key (so each failing row can be identified).

| Flag | Description |
|---|---|
| `--col NAME` | Filter to a single computed column |

```bash
pcli errors my_dir/my_table
pcli errors my_dir/my_table --col embedding
```

## Mutation commands

Every mutation accepts the universal `-f`/`--force`, `-n`/`--dry-run`, and `--json` flags. With a TTY they prompt `[y/N]`; in non-interactive contexts they refuse to proceed without `-f`.

### `pcli drop`

Drop a table or view. Use `pcli rm` for directories.

| Flag | Description |
|---|---|
| `--cascade` | Also drop dependent views |

```bash
pcli drop my_dir/my_table -f                 # drop a table
pcli drop my_dir/my_table --cascade -f       # also drop dependent views
pcli drop my_dir/my_table -n                 # dry-run
```

### `pcli rm`

Remove a directory. Use `pcli drop` for tables/views.

| Flag | Description |
|---|---|
| `-r`, `--recursive` | Also remove contained tables/views/subdirectories |

```bash
pcli rm my_dir -f                  # remove an empty directory
pcli rm my_dir -r -f               # recursive: also remove contained tables/subdirs
```

### `pcli rename`

Rename in place; the parent directory is preserved. `<new_name>` must be a single leaf name (no `/` or `.`). Takes only universal flags.

```bash
pcli rename my_dir/old_name new_name
```

### `pcli mv`

Move a table/view/dir under a different directory; the leaf name is preserved. `<new_dir>` can be `''` or `/` for the root directory. Takes only universal flags.

```bash
pcli mv my_dir/my_table other_dir              # -> other_dir/my_table
pcli mv my_dir/my_table /                      # move to root
```

### `pcli revert`

Undo recent ops on a table. Each revert undoes one op; `--steps` repeats.

| Flag | Description |
|---|---|
| `--steps N` | Number of consecutive reverts (default 1) |

```bash
pcli revert my_dir/my_table -f                 # undo the last op
pcli revert my_dir/my_table --steps 3 -f       # roll back 3 versions
```

<Warning>
Revert is irreversible. Run `pcli history my_dir/my_table` first to see what would be undone.
</Warning>

## Interactive shell

For agentic or scripted workloads that issue many commands in sequence, `pcli shell` amortizes Python startup over the session:

```text
$ pcli shell
pcli> ls
path                kind
agent_demo          dir
chess_vids          table
...
pcli> describe chess_vids
...
pcli> exit
```

Inside the shell, every pcli command is available unmodified. Errors from one command don't kill the session. Use `help`, `exit`, `quit`, or Ctrl-D to leave.

## Output and scripting

Most commands accept `--json` for stable, machine-readable output (exceptions: `shell` is interactive, `health` is already JSON):

```bash
pcli ls --json | jq '.entries[] | select(.kind == "table")'
pcli get my_dir/my_table 42 --json | jq '.row'
pcli count my_dir/my_table --json | jq '.count'
```

Without `--json`, output is column-aligned text.
